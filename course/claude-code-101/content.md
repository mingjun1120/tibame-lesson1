# 認識：Claude Code 是什麼
> Anthropic 官方的終端機 AI 編程代理 — 讓 Claude 直接在你的專案裡讀檔、改碼、跑指令

## 不只是聊天，而是會動手的 Agent

### 🤖 從「給建議」到「幫你做」
- 一般 AI 聊天視窗：你問、它答，複製貼上靠自己
- Claude Code：直接讀你的檔案、修改程式碼、執行指令、跑測試
- 你下達目標，它自己規劃步驟、動手完成、回報結果

### 🧭 它住在你最熟悉的地方
- 直接在終端機（Terminal）運行，貼著你的專案根目錄
- 也能裝進 VS Code / JetBrains，用側邊欄對話
- 還有桌面 App 與網頁版（claude.ai/code），多端共用同一套能力

### ⚡ 適合什麼樣的人
[tags]
- [green] 想用 AI 加速開發的工程師
- [blue] 想讓 AI 跑自動化任務的人
- [purple] 不想離開終端機的命令列愛好者
- [orange] 想把重複工作流封裝起來的人
[/tags]

> **CLI 不是限制，而是力量**
> 把 AI 放進終端機，意味著它能直接觸碰你的真實環境 — 檔案、git、測試、套件管理。
>
> 這讓 Claude Code 從「給你建議的顧問」變成「幫你完成工作的同事」。

# 安裝：五分鐘跑起第一個對話
> 確認 Node.js → 安裝 Claude Code → 登入 → 信任資料夾，就能開始

## 安裝前的準備

### 📦 確認 Node.js
- Claude Code 透過 npm 安裝，需要 Node.js 18 以上
- 在終端機輸入 `node -v` 確認版本
- 沒看到版號就先安裝 Node.js（建議用 nvm 管理多版本）

```prompt [label="確認 Node 版本"]
node -v
```

## 安裝與登入

### ⬇️ 全域安裝 Claude Code

```prompt [label="安裝指令"]
npm install -g @anthropic-ai/claude-code
```

### 🚀 第一次啟動
- 先 `cd` 進你的專案資料夾，再輸入 `claude` 啟動
- 第一次會引導你登入（Claude 訂閱帳號或 Anthropic API Key）
- 詢問是否信任這個資料夾，選 Yes 才能繼續

```prompt [label="啟動"]
cd my-project
claude
```

[flow]
1. 登入帳號 — 用 Claude 訂閱或 Anthropic API Key
2. 信任資料夾 — 確認 AI 的工作範圍
3. 開始對話 — 把終端機當成與 AI 的對話視窗
[/flow]

> **啟動的位置，就是它的工作範圍**
> Claude Code 讀寫檔案、跑指令都以啟動時的資料夾為界。
>
> 養成習慣：先 `cd` 到正確的專案再啟動，別讓它在 home 目錄亂跑。

# 操作：和 AI 一起寫程式
> 下目標、看它規劃、審核它的修改 — 這就是 Claude Code 的核心循環

## 基本對話循環

### 💬 用自然語言下達任務
- 不需要背指令，直接用白話描述你要什麼
- 它會自己決定讀哪些檔、改哪些碼、跑什麼指令
- 邊做邊回報：你能即時看到它動了哪些檔、哪幾行

```prompt [label="一句話交辦任務"]
幫我看看 src/auth 底下的登入流程，
然後加上「記住我」的勾選框，把偏好存到 localStorage
```

### ✅ 它會邊做邊回報
- 讀檔、改檔、跑指令前都會顯示它的動作
- 跑完會告訴你結果：測試過了沒、有沒有錯誤
- 不滿意可以直接接著對話請它調整

## Plan Mode：先想清楚再動手

### 🗺️ 讓 AI 先規劃、再執行
- 複雜任務先進 Plan Mode：AI 只研究與規劃、不改檔
- 你審核計畫後，再讓它實際執行
- 適合大型重構、跨多檔案的功能開發

> **審核，是你最重要的工作**
> AI 寫程式很快，但「按下同意」的責任在你身上。
>
> 養成審核每個修改的習慣 — 不是因為 AI 常出錯，而是因為理解程式碼的人，永遠是專案真正的負責人。

[lab-session title="🛠️ 動手做：你的第一個任務" duration="10 分鐘" hint="卡關很正常，把錯誤訊息直接貼回去問 AI 就對了"]
- 在任一個小專案資料夾啟動 `claude`
- 用一句白話請它新增一個小功能或修一個 bug
- 觀察它讀了哪些檔、改了哪幾行，再決定要不要同意
[/lab-session]

# 記憶：用 CLAUDE.md 教 AI 你的專案規矩
> 把專案慣例寫進 CLAUDE.md，AI 每次對話都會自動讀取

## CLAUDE.md 是什麼

### 📝 專案的長期記憶
- 放在專案根目錄的 `CLAUDE.md`，每次啟動自動載入
- 寫下技術棧、程式風格、命名慣例、不要做的事
- AI 會遵守，就像新同事讀過你的團隊規範

```prompt [label="自動生成 CLAUDE.md"]
/init
```

### 🎯 該寫什麼進去
[tags]
- [blue] 技術棧與框架版本
- [green] 程式風格與命名慣例
- [purple] 常用指令（build / test / lint）
- [orange] 絕對不要碰的檔案或目錄
[/tags]

```code [label="CLAUDE.md 範例"]
# 專案規範

- 一律用繁體中文回覆
- 前端：React + TypeScript，樣式用 Tailwind
- 提交前必跑 `npm run lint && npm test`
- 不要修改 src/generated/ 底下的自動產生檔
```

> **與其每次提醒，不如寫成規則**
> 如果你發現自己一直在跟 AI 重複同一句叮嚀，那就該把它寫進 CLAUDE.md。
>
> 好的 CLAUDE.md 讓 AI 的第一次輸出就接近你要的樣子，省下反覆來回。

# 擴充：斜線指令、Skills 與 MCP
> 從內建指令到自訂工作流，把 Claude Code 變成你的專屬工具

## 斜線指令（Slash Commands）

### ⌨️ 用 / 呼叫功能
- 在輸入框打 `/` 會列出所有可用指令
- 內建好用指令：`/init`、`/clear`、`/config`、`/review`
- 也能自訂屬於你團隊的專屬指令

[flow]
1. /init — 初始化專案的 CLAUDE.md
2. /clear — 清空對話脈絡，開始全新任務
3. /review — 審查當前的程式碼變更
[/flow]

## Agent Skills

### 🧩 把工作流封裝成可重複使用的技能
- Skill＝一包指示＋腳本，讓 AI 遇到特定情境自動觸發
- 例如：把講稿轉成課程網頁、把音檔轉成字幕
- 設計一次，之後一句話就能重現整套流程

> **Skill 是把「怎麼做」交給 AI 記住**
> 你不必每次都把完整步驟貼給 AI。把流程寫成 Skill，AI 會在對的時機自己拿出來用。
>
> 這是 Vibe Coding 從「一次性 Demo」走向「長期可用工具」的關鍵。

## MCP：連接外部世界

### 🔌 讓 AI 連上你的工具
- MCP（Model Context Protocol）讓 Claude Code 接上外部服務
- 例如 GitHub、資料庫、Notion、Slack
- AI 就能查 issue、跑 SQL、讀文件，不再侷限於本機檔案

# 自動化：Hooks 與 Git 工作流
> 讓固定動作自動發生，讓 commit 與 PR 由 AI 代勞

## Hooks：自動觸發的行為

### 🪝 在特定時機自動執行
- Hook 是「每次發生 X 就做 Y」的設定
- 例如：每次 AI 改完檔，就自動跑 formatter
- 由 harness 執行，不靠 AI 自律，所以更可靠

## Git 工作流

### 🔀 讓 AI 幫你 commit 與開 PR
- 請 AI 分析變更，拆成乾淨的多個 commit
- 自動產生符合慣例的 commit message 與 PR 描述
- 你專注審核邏輯，瑣事交給 AI

```prompt [label="請 AI 提交"]
幫我把這次變更拆成幾個合理的 commit 並提交
```

> **降低認知負擔，而不是放棄把關**
> AI 讓寫程式變快，也讓 Code Review 的負擔變重。
>
> 讓 AI 拆乾淨的 commit、寫清楚的 PR — 是為了讓「審核」這件事還做得動。

# 心法：用對的姿勢使用 Claude Code
> 工具會變，但善用 AI 的思維方式一直管用

## 三個讓你事半功倍的習慣

### 🧹 管理你的脈絡（Context）
- 切換任務時用 `/clear` 開新對話，避免脈絡污染
- 對話太長時 Claude Code 會自動摘要，不用怕中斷
- 一次專注一件事，AI 的表現最好

### 🎯 把需求講清楚
- 給目標、給限制、給範例，遠勝一句模糊的話
- 不確定時，讓 AI 先進 Plan Mode 或反問你
- 把你腦中的「驗收標準」直接說出來

### 🛡️ 保持驗證的習慣
- 跑得起來不等於做對了，讓 AI 跑測試、自己也看結果
- 重要邏輯一定要審，別把責任全交給 AI
- AI 是加速器，不是免責聲明

# Q&A

[qa-session title="新手常見問題"]
- Q: 完全不會寫程式，可以用 Claude Code 嗎？
  A: 可以入門，但建議至少懂基本的終端機操作與專案結構。它會幫你動手，但「審核它做得對不對」需要你逐步累積判斷力。
- Q: 一定要付費嗎？
  A: 需要 Claude 訂閱或 Anthropic API Key。用 API Key 會依實際用量計費，記得先設好用量上限，避免帳單超出預期。
- Q: 它會不會亂改我的檔案？
  A: 每個修改與指令在執行前都會顯示出來讓你確認。複雜任務先用 Plan Mode 看計畫，再決定是否放行，主控權一直在你手上。
- Q: 和在網頁版聊天有什麼不同？
  A: 最大差別是它能直接接觸你的真實環境 — 讀寫檔案、跑 git、執行測試，而不只是給你一段要自己複製貼上的程式碼。
[/qa-session]
