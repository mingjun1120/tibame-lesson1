# 提案版型範本

複製下方 frontmatter 與六大區塊，貼進 `<提案資料夾>/README.md` 後填寫。
六大區塊**順序固定**、用 `##` 標題，不可增刪或改名。

---

```markdown
---
course_title: 課程名稱
client:                       # 留空 → build 時自動採用上層資料夾名稱
duration: 待確認               # 例：2 次 × 6 小時（共 12 小時）
date: 2026-05-22
status: 提案中                 # 提案中 / 洽談中 / 已確認 / 已結案
cover: false                   # true 啟用封面頁；可用 --cover/--no-cover 旗標覆寫
---

## 課程簡介

<!-- 200 字以內。課程定位、解決什麼痛點、學員上完能帶走什麼。 -->

## 使用工具

- 工具 A — 一句話用途
- 工具 B — 一句話用途

## 預計成效

- 能獨立完成 ⋯⋯
- 能 ⋯⋯

## 適合對象

- ⋯⋯
- ⋯⋯

## 課程目錄

1. 章節一
2. 章節二

## 課程大綱

### 章節一

- 重點
- 重點，可用空白縮排再展開細節：
  - 子項目
  - 子項目

### 章節二

- 重點
```

---

## 欄位說明

| frontmatter 欄位 | 說明 |
|------------------|------|
| `course_title` | 課程名稱，PDF 第一頁大標 |
| `client` | 客戶名稱；留空則自動採用提案資料夾名稱 |
| `duration` | 時數；不確定填 `待確認`，不要編造 |
| `date` | 提案日期 |
| `status` | 提案進度 |
| `cover` | `true` / `false`；啟用後第一頁為正式封面，內頁從第二頁開始。可被 CLI 旗標 `--cover` / `--no-cover` 覆寫 |

> **講師介紹不要寫進 README.md** —— build 時會自動從 `config.yaml` 注入 PDF 最後的「關於講師」區塊。

---

## 多場次課程

課程分多場（多天／多梯次）時，在「課程目錄」與「課程大綱」用 `=== 場次標題 ===` 插入場次分隔帶；模組仍用 `###`，課程目錄與課程大綱的各場次編號都會獨立從 1 起算（課程大綱的章節編號由 PDF 自動加上，毋須在 Markdown 內手寫）。

```markdown
## 課程目錄

=== 第 1 場：上午場 ===

1. 模組一　……
2. 模組二　……

=== 第 2 場：下午場 ===

1. 模組三　……

## 課程大綱

=== 第 1 場：上午場 ===

### 模組一　……

- 重點

=== 第 2 場：下午場 ===

### 模組三　……

- 重點
```

## config.yaml 範例

`config.yaml` 放在提案根目錄（所有提案共用），`build-pdf.mjs` 會從提案資料夾往上層自動尋找。

```yaml
# 提案文件共用設定 — 講師介紹、品牌色、頁尾
instructor:
  name: 講師姓名
  tagline: 一句話頭銜
  bio: >
    講師介紹段落，會渲染在 PDF 最後的「關於講師」區塊。
  highlights:                             # 條目可用 [文字](網址) 加超連結
    - 📚 重點成就一
    - 🎤 重點成就二
  links:
    - Medium｜https://example.com        # 格式為「標籤｜網址」，分隔符是全形｜
    - YouTube｜https://example.com

brand:
  accent: "#2563EB"                       # 主色，用於標題與分隔線

footer: 講師姓名 · 課程提案                # 每頁頁尾左側文字
```
