# ShareBox 社區共享工具平台 — 需求規格書

文件版本：v1.0　｜　發佈日期：2026-02-12　｜　撰寫：專案經理 林佳穎

## 1. 文件目的

本文件定義 ShareBox 社區共享工具平台第一版（MVP）的功能需求與非功能性需求，作為設計與開發的依據。商業背景請參閱《商業企劃書》，技術實作請參閱《技術架構說明》。

## 2. 產品定位與目標使用者

- 產品定位：以「社區」為單位的工具共享行動 App。
- 目標使用者：都會區公寓大廈住戶，特別是雙北、桃園、台中的新型社區大樓住戶。
- 使用情境：住戶想短期使用某項工具（如電鑽）但不想購買，透過 App 向同社區鄰居借用。

## 3. 使用者角色

- 借用人：搜尋並申請借用工具的住戶。
- 出借人：刊登閒置工具供他人借用的住戶（同一使用者可同時是兩種角色）。
- 平台管理員：處理檢舉、管理使用者、檢視營運數據。

## 4. 功能需求清單

| 編號 | 功能名稱 | 說明 | 優先級 |
|---|---|---|---|
| F-01 | 手機 OTP 註冊／登入 | 以手機號碼接收簡訊驗證碼完成註冊與登入 | Must |
| F-02 | 社區綁定 | 依居住地址驗證並綁定所屬社區 | Must |
| F-03 | 工具刊登 | 拍照、選擇分類、填寫描述、設定押金與租金 | Must |
| F-04 | 附近工具搜尋 | 以地圖與清單依分類、距離搜尋附近工具 | Must |
| F-05 | 工具詳情與收藏 | 檢視工具詳細資訊並加入收藏 | Must |
| F-06 | 借用申請與審核 | 借用人提出申請，出借人同意或拒絕 | Must |
| F-07 | 訂單即時聊天 | 借用雙方於訂單內即時通訊約定時間地點 | Must |
| F-08 | 押金代管與金流 | 透過 ECPay 進行押金代管與付款 | Must |
| F-09 | 歸還確認與評價 | 確認歸還並互相評分評價 | Must |
| F-10 | 信用評分機制 | 依交易與評價紀錄計算使用者信用分數 | Should |
| F-11 | 平台保險與自動理賠 | 工具遺失或損壞時的保險與理賠機制 | Could |
| F-12 | 推播通知 | 申請、同意、歸還提醒等即時通知 | Must |
| F-13 | 個人中心 | 我的刊登、我的借用、信用分數總覽 | Must |
| F-14 | 後台管理 | 檢舉處理、使用者管理、數據儀表板 | Should |

## 5. 功能優先級（MoSCoW）

- Must have（第一版必備）：F-01、F-02、F-03、F-04、F-05、F-06、F-07、F-08、F-09、F-12、F-13。
- Should have（第一版盡量納入）：F-10、F-14。
- Could have（資源允許再做）：F-11。
備註：F-10、F-11 的最終排程於專案執行期間可能依開發進度調整，實際範圍以最新一次會議決議為準。

## 6. 使用者故事（節錄）

- 身為一位社區住戶，我想搜尋附近鄰居的電鑽，這樣我就不必為了組一次家具而購買。
- 身為一位工具擁有者，我想設定押金，這樣工具損壞或遺失時我有保障。
- 身為一位借用人，我想在 App 內直接與出借人聊天約時間，這樣我不需交換私人聯絡方式。
- 身為一位出借人，我想看到借用人的信用評分，這樣我能判斷是否要出借。
- 身為一位平台管理員，我想處理使用者檢舉，這樣能維持平台的安全與信任。

## 7. 非功能性需求

| 類別 | 需求內容 |
|---|---|
| 效能 | 工具搜尋 API 回應時間 P95 小於 800 毫秒 |
| 可用性 | 服務可用率達 99.5% 以上 |
| 安全性 | 所有 API 走 HTTPS；密碼與 Token 加密；個資符合《個人資料保護法》 |
| 相容性 | 支援 iOS 15 以上、Android 10 以上 |
| 可擴充性 | 單一社區規模達 2,000 戶不需重構 |
| 在地化 | 介面為繁體中文，金額單位為新台幣 |

## 8. 不在第一版範圍

跨社區借用、企業（管委會）方案、在地商家廣告，列為未來版本評估。
