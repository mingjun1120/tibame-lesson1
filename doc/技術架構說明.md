## 1. 專案技術概述

ShareBox 是一款以「社區」為單位的工具共享行動 App，讓同一社區的住戶能安全地互借閒置工具與裝備（如電鑽、梯子、吸塵器、露營裝備）。

本文件聚焦於**技術面**，說明系統如何實作。產品需求與功能清單請見《需求規格書》，商業面與市場分析請見《商業企劃書》。

- 平台型態：行動 App（iOS / Android）+ 雲端後端 API
- 開發語言：前端 TypeScript（React Native）、後端 Python
- 程式碼託管：GitHub（私有 repo）
- 開發團隊：後端 1 人、前端／App 1 人、UI/UX 1 人

## 2. 技術選型與理由

| 層級 | 採用技術 | 選用理由 |
|------|----------|----------|
| 前端 / App | React Native 0.74 + Redux Toolkit | 單一程式碼庫同時支援 iOS／Android，前端僅 1 人，需最大化人力效益 |
| 後端 API | Python 3.12 + FastAPI | 開發速度快、原生 async、自動產生 OpenAPI 文件 |
| 主資料庫 | PostgreSQL 16（含 PostGIS） | 關聯完整性佳，PostGIS 支援「附近工具」地理查詢 |
| 快取 | Redis 7 | 快取熱門搜尋結果與使用者 Session，降低 DB 負載 |
| 雲端平台 | AWS | 團隊熟悉度高、服務完整、可隨用戶成長擴充 |
| 推播通知 | Firebase Cloud Messaging | 跨平台推播、免費、穩定 |
| 金流 | 綠界科技 ECPay | 支援台灣信用卡與「押金代管」情境 |
| 地圖服務 | Google Maps Platform | 地圖顯示、地理編碼、附近地點搜尋 |
| 錯誤監控 | Sentry + AWS CloudWatch | 即時掌握當機與效能問題 |

## 3. 系統架構

文字示意圖：

```text
[ iOS App ]      [ Android App ]
      \              /
       \            /   (HTTPS / JWT)
       [ CloudFront CDN ]
              |
      [ AWS ECS Fargate ]  <-->  [ Redis 快取 ]
       FastAPI 後端服務
              |
       [ RDS PostgreSQL ]
              |
   +----------+--------------+-----------------+
[ S3 圖片儲存 ]  [ ECPay 金流 ]  [ Google Maps API ]
                              [ Firebase FCM 推播 ]
```

- App 透過 HTTPS 呼叫後端 API，使用 JWT 進行身分驗證。
- 後端以容器化方式部署於 AWS ECS Fargate，可水平擴充。
- 工具照片、使用者頭像上傳至 S3，透過 CloudFront 加速。
- 金流交易委由 ECPay 處理，平台不直接保存信用卡資料。

## 4. 資料模型

主要資料表（實體）：

| 實體 | 說明 | 主要欄位 |
|------|------|----------|
| User | 使用者 | id、暱稱、手機、email、信用分數、所屬社區、註冊時間 |
| Community | 社區 | id、名稱、地址、行政區、經緯度 |
| Tool | 工具刊登 | id、擁有者、名稱、分類、描述、照片、押金、租金、狀態 |
| BorrowOrder | 借用訂單 | id、工具、借用人、出借人、起訖日、押金、訂單狀態 |
| Review | 評價 | id、訂單、評分、評論、評價對象 |
| ChatMessage | 訂單聊天訊息 | id、訂單、發送者、內容、時間 |
| Payment | 金流紀錄 | id、訂單、金額、類型、ECPay 交易序號、狀態 |

關聯重點：

- 一位 User 屬於一個 Community；一個 Community 有多位 User。
- 一位 User 可刊登多個 Tool；一個 Tool 可產生多筆 BorrowOrder。
- 一筆 BorrowOrder 對應一組借用人／出借人，並關聯多筆 ChatMessage 與 Payment。

## 5. 主要 API（節錄）

| 方法 | 路徑 | 說明 |
|------|------|------|
| POST | /api/auth/otp | 發送手機簡訊 OTP |
| POST | /api/auth/verify | 驗證 OTP 並登入，回傳 JWT |
| GET | /api/tools?lat=&lng=&radius=&category= | 依位置與分類搜尋附近工具 |
| POST | /api/tools | 刊登工具 |
| GET | /api/tools/{id} | 取得工具詳情 |
| POST | /api/orders | 建立借用申請 |
| PATCH | /api/orders/{id} | 更新訂單狀態（同意／拒絕／歸還） |
| POST | /api/payments/deposit | 押金代管請款 |
| GET | /api/users/{id}/reviews | 取得使用者評價 |

完整 API 文件由 FastAPI 自動產生，部署後可於 `/docs` 路徑檢視。

## 6. 部署與 CI/CD

- 版本控管：GitHub，採 `main` / `develop` / feature branch 流程。
- CI/CD：GitHub Actions，推送至 `develop` 自動執行測試與建置；合併至 `main` 後自動部署至 AWS。
- 容器映像：建置後推送至 AWS ECR，由 ECS Fargate 拉取部署。
- 環境區分：`dev`（開發）、`staging`（測試）、`production`（正式）三套環境。

## 7. 開發環境設定

開發人員初次設定步驟：

1. 安裝 Node.js 20、Python 3.12、Docker。
2. `git clone` 專案後，後端執行 `pip install -r requirements.txt`。
3. 前端於 `app/` 目錄執行 `npm install`。
4. 複製 `.env.example` 為 `.env`，填入本機資料庫與第三方服務金鑰。
5. 以 `docker compose up` 啟動本機 PostgreSQL 與 Redis。
6. 後端執行 `uvicorn main:app --reload`，前端執行 `npx react-native run-ios` 或 `run-android`。

## 8. 技術相關注意事項

- 信用評分機制（F-10）與平台保險自動理賠（F-11）已於 2026-04 規劃為 **Phase 2** 項目，MVP 階段先以人工審核的簡易押金代管處理。決策背景詳見《開發進度與技術決策紀錄》ADR-004。
- 個資處理須符合《個人資料保護法》，使用者手機與身分資料加密儲存。
- 所有對外 API 一律走 HTTPS。
